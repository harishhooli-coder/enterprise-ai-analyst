Ask a Question

Like asking a reference librarian for a pre-approved fact sheet — not for a guess.

A librarian, not an improviser

Imagine you walk up to a reference desk and ask: What was total revenue last month? You do not want the librarian to invent numbers from memory. You want them to pull a vetted fact sheet from a trusted shelf.

That is what the Data-Answers Agent does for business users. You type a plain-English question in plain language. The system looks up a pre-approved answer path — never free-form SQL improvisation — and returns a number you can trust.

Two pieces work together: data-answers-ui (the Test Console you click in) and data-answers-agent (the Python backend that does the heavy lifting).

What you actually do in AskConsole

Open the Test Console (AskConsole in the Next.js UI). Type a question like What was total revenue last month?, pick a user ID and regions, then hit Ask.

That single click kicks off a chain: your browser packages the question and sends it to the backend. Nothing magic — just a structured message over the network.

The code behind the Ask button

When you click Ask, AskConsole calls askQuestion in data-answers-ui/lib/api.ts. These four lines are the handoff from UI to backend.

api.ts 
export async function askQuestion(body: AskRequest): Promise<ApiResponse> {
  return request<ApiResponse>("/ask", {
    method: "POST",
    body: JSON.stringify(body),
  });
}
Plain English
Line 39 — Export a function named askQuestion. It takes the question plus user info and promises to return a typed response envelope.
Line 40 — Call the shared request helper, targeting the endpoint /ask on the FastAPI server.
Line 41 — Use the POST method so the question travels in the request body, not the URL.
Line 42 — Turn the JavaScript object into a JSON string so the backend can read question and user fields reliably.
